docker 常用命令

2020-07-20
工具
  • docker

镜像管理

配置镜像仓库的国内加速地址

# Ubuntu16.04+、Debian8+、CentOS7适用
# 创建或修改daemon配置文件:/etc/docker/daemon.json,内容如下
{
  "registry-mirrors": ["https://<个人ID>.mirror.aliyuncs.com"]
}
# 保存文件后，重启服务
sudo systemctl daemon-reload
sudo systemctl restart docker

从云端镜像仓库获取新镜像

# 安装最新版镜像
docker pull <镜像名>

# 指定镜像版本安装
docker pull <镜像名>:<镜像版本>

容器管理

从镜像创建并运行一个新容器

# 前台运行
docker run <镜像名>

# 后台运行
docker run -d <镜像名>

# 把容器的一个服务端口映射到宿主机上
docker run -p <宿主机端口>:<容器端口> <镜像名>

# 映射多个端口
docker run -p <宿主机端口1>:<容器端口1> \
			-p <宿主机端口2>:<容器端口2> \
			-p <宿主机端口3>:<容器端口3> <镜像名>
			
# 指定容器名
docker run --name <容器名> <镜像名>

# 以交互模式运行,在容器内执行 /bin/bash 命令
docker run -it <镜像名> /bin/bash

管理已创建的容器

# 列出运行中的容器,以下二者都可以
docker container ls
docker ps

# 列出所有已创建的容器，包括停止运行的
docker container ls -a
docker ps -a

# 启动 | 停止 | 重启 | 删除容器
docker [ start | stop | restart | rm ] <容器名或容器ID>

# 重命名容器
docker rename <原容器名> <新容器名>

# 交互模式在容器内执行 /bin/bash 命令
docker run -it <容器名或容器ID> /bin/bash

# 从宿主机拷贝文件或文件夹到容器
docker cp <宿主机中的文件路径> <容器名或容器ID>:<容器内文件路径>

# 从容器拷贝文件或文件夹到宿主机
docker cp <容器名或容器ID>:<容器内文件路径> <宿主机中的文件路径>

Docker Compose

使用 Docker Compose 可以很方便地定义和运行多容器应用程序，通过yml文件即可进行配置。

# 首先移动到docker-compose.yml配置文件所在目录
cd <docker-compose.yml所在目录>

# 前台运行应用程序
docker-compose up

# 后台运行应用程序
docker-compose up -d

# 停止应用程序
docker-compose stop

# 重新启动应用程序
docker-compose restart

# 删除停止运行的容器
docker-compose rm

# 停止运行应用程序并删除对应容器
docker-compose down
